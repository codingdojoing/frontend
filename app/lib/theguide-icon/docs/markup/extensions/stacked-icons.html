Stacked icons consist of two or more icons 'stacked' on top of each other, within a container with class stacked-icon. The size of the icon is also set on the container, for instance icon-jb (See icon sizes).

The stacked icon class names consist of two parts:

  1. The stacked icon name, for instance icon-target, and
  2. the stacking order appendix. The bottom icon has appendix -z1, the next icon -z2, etc.

The <span> elements must be placed according to the stacking order: -z1 first, -z2 second, etc.

Set to block-level

By default, the stacked icon container is set to display: inline-block. This will normally result in a slight bottom spacing. If you need to eliminate said spacing, for instance within a table cell, or for other reasons, add class icon-block to the container, which will set it to display: block.

Setting the stacked icon container to display: block means that it will take up the full width of its own container, so it will no longer be possible to place any text beside it within the same container.

Inline-block VS Block

By default the stacked icon is display: inline-block

When an icons needs to be placed as a block element, display: block, for example in a table cell so there will be no bottom spacing, add the class .icon-block to the stacked icon.
